# Deploy `k8shelper` on kagent (Anthropic + Gemini)

This lab deploys the `k8shelper` agent (a Kubernetes helper using Google ADK + LiteLLM) to the **kagent** runtime you registered in [051](051-kagent-provider.md). Two image-backed flavors are provided:

| Flavor | Model | Image asset |
|---|---|---|
| Anthropic / Claude | `claude-sonnet-4-6` | [`assets/k8shelper-anthropic/`](assets/k8shelper-anthropic/) |
| Gemini | `gemini-3.5-flash` | [`assets/k8shelper-gemini/`](assets/k8shelper-gemini/) |

The pattern is the same for both:

1. Build and push the image
2. Apply the AgentRegistry `Agent` referencing the image
3. Create a Kubernetes Secret for the model API key in the `kagent` namespace
4. Apply the AgentRegistry `Deployment` targeting the kagent Runtime
5. Patch the generated `kagent.dev/v1alpha2 Agent` CR to mount the Secret

## Lab Objectives

- Build a `linux/amd64` image and push to a registry your cluster can pull from
- Register the `k8shelper` Agent + Deployment via `arctl`
- Patch the generated kagent CR to inject the model API key Secret
- Verify the pod reaches `Ready=True`

## Prerequisites

- [051 — kagent Runtime registered](051-kagent-provider.md)
- A container registry your cluster can pull from
- An Anthropic API key (`ANTHROPIC_API_KEY`) or Google Gemini API key (`GOOGLE_API_KEY`)
- Docker with `buildx`

---

## Anthropic Variant (Claude)

### 1. Build and Push the Image

```bash
cd assets/k8shelper-anthropic

# Build linux/amd64 (cross-compiles on Apple Silicon)
docker buildx build --platform linux/amd64 \
  -t k8shelperanthropic:claude-sonnet-4-6 --load .

# Tag + push to your registry
export K8SHELPER_IMAGE="<your-registry>/k8shelperanthropic:claude-sonnet-4-6"
docker tag k8shelperanthropic:claude-sonnet-4-6 "${K8SHELPER_IMAGE}"
docker push "${K8SHELPER_IMAGE}"
```

The `Dockerfile` is **autogenerated by the AgentRegistry CLI** (`arctl init` for a BYO agent). It bases on `ghcr.io/kagent-dev/kagent/kagent-adk:0.8.0-beta6`. If you need to regenerate it, do so via the CLI rather than editing it by hand.

### 2. Apply the Agent

```bash
# From the workshop root; image is set directly in the manifest
arctl apply -f assets/providers/kagent/anthropicagent/k8shelperanthropic.yaml
arctl get agents
```

If you used a different image path, edit the `spec.source.image` field in the manifest before applying.

### 3. Create the API-Key Secret

```bash
kubectl create secret generic k8shelper-anthropic \
  -n kagent \
  --from-literal=ANTHROPIC_API_KEY="${ANTHROPIC_API_KEY}" \
  --dry-run=client -o yaml | kubectl apply -f -
```

### 4. Apply the Deployment

```bash
arctl apply -f assets/providers/kagent/anthropicagent/ardeploy.yaml
```

> **Order matters.** The `Agent` and any `deploymentRefs` (e.g., the MCP deployment in [072](072-wire-mcp-to-agent.md)) must exist in the catalog **before** you apply the `Deployment`, or you'll get `spec.targetRef: referenced resource not found` / `spec.deploymentRefs[0]: referenced resource not found`.

### 5. Patch the kagent Agent CR to Inject the Secret

AgentRegistry's `Deployment.spec.env` supports literal values only. Secrets must be patched onto the generated kagent CR:

```bash
kubectl patch agent k8shelperanthropic -n kagent --type='json' -p='[
  {"op":"add","path":"/spec/byo/deployment/env/-","value":{"name":"ANTHROPIC_API_KEY","valueFrom":{"secretKeyRef":{"name":"k8shelper-anthropic","key":"ANTHROPIC_API_KEY"}}}}
]'

kubectl rollout status deployment/k8shelperanthropic -n kagent --timeout=5m
```

> Direct patches to generated kagent resources can be overwritten by a future AgentRegistry redeploy. If you need this to survive a redeploy, mount the Secret via the Runtime's BYO env config (currently a kagent-side concern) or re-run the patch in your CI/CD.

### 6. Verify

```bash
kubectl get agents.kagent.dev -n kagent k8shelperanthropic -o yaml
kubectl get pods -n kagent -l kagent=k8shelperanthropic
kubectl get svc -n kagent -l kagent=k8shelperanthropic
kubectl get deploy k8shelperanthropic -n kagent -o yaml \
  | grep -E 'MODEL_NAME|MODEL_PROVIDER|ANTHROPIC_API_KEY|image:'
```

You should see `MODEL_PROVIDER=anthropic`, `MODEL_NAME=claude-sonnet-4-6`, the `ANTHROPIC_API_KEY` Secret reference, and your image.

### Model Configuration Notes

The agent uses ADK with LiteLLM for Anthropic, so it calls Anthropic **directly** with `ANTHROPIC_API_KEY` (not via Vertex). When `MODEL_PROVIDER=anthropic`, the agent code in [`assets/k8shelper-anthropic/k8shelper/agent.py`](assets/k8shelper-anthropic/k8shelper/agent.py) prefixes `MODEL_NAME` with `anthropic/` for LiteLLM if you pass a bare Claude model name.

## Gemini Variant

### 1. Build and Push the Image

```bash
cd assets/k8shelper-gemini

docker buildx build --platform linux/amd64 \
  -t k8shelpergemini:gemini-3.5-flash --load .

export K8SHELPER_IMAGE="<your-registry>/k8shelpergemini:gemini-3.5-flash"
docker tag k8shelpergemini:gemini-3.5-flash "${K8SHELPER_IMAGE}"
docker push "${K8SHELPER_IMAGE}"
```

### 2. Apply the Agent (envsubst the image)

```bash
envsubst < assets/providers/kagent/geminiagent/k8shelpergemini.yaml | arctl apply -f -
arctl get agents
```

### 3. Create the API-Key Secret

```bash
kubectl create secret generic k8shelper-google \
  -n kagent \
  --from-literal=GOOGLE_API_KEY="${GOOGLE_API_KEY}" \
  --dry-run=client -o yaml | kubectl apply -f -
```

### 4. Apply the Deployment

```bash
arctl apply -f assets/providers/kagent/geminiagent/ardeploy.yaml
```

### 5. Patch the kagent Agent CR

```bash
kubectl patch agent k8shelper -n kagent --type='json' -p='[
  {"op":"add","path":"/spec/byo/deployment/env/-","value":{"name":"GOOGLE_API_KEY","valueFrom":{"secretKeyRef":{"name":"k8shelper-google","key":"GOOGLE_API_KEY"}}}}
]'

kubectl rollout status deployment/k8shelper -n kagent --timeout=5m
```

### 6. Verify

```bash
kubectl get agents.kagent.dev -n kagent k8shelper -o yaml
kubectl get pods -n kagent -l kagent=k8shelper
kubectl get svc -n kagent -l kagent=k8shelper
kubectl get deploy k8shelper -n kagent -o yaml \
  | grep -E 'MODEL_NAME|MODEL_PROVIDER|GOOGLE_API_KEY|image:'
```

> The agent reads `MODEL_NAME` from the environment and defaults to `gemini-3.5-flash`. An older demo image that hardcoded `gemini-2.0-flash` will hit `429 RESOURCE_EXHAUSTED` — rebuild with the current source.

## Troubleshooting

```bash
arctl get deployment k8shelperanthropic-kagent -o yaml
arctl get deployment k8shelper-kagent -o yaml
```

| Symptom | Fix |
|---|---|
| `spec.targetRef: referenced resource not found` | You applied the `Deployment` before the `Agent`. Apply the `Agent` first. |
| `spec.deploymentRefs[0]: referenced resource not found` | The MCP server deployment you reference doesn't exist yet. See [072](072-wire-mcp-to-agent.md). |
| `authentication token expired during deployment; please retry` | kagent controller is rejecting the forwarded identity. See [051 step 3](051-kagent-provider.md#3-verify-the-kagent-api-accepts-requests-from-agentregistry). |
| Image pull failures | Image is private — wire image pull secrets into the kagent namespace, or push to a registry the nodes can pull from anonymously. |
